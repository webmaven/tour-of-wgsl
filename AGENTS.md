# Tour of WGSL - Agent Instructions

## Overview

This repository contains the source code for a static site generator that serves a Tour of WGSL (WebGPU Shading Language). The site is built using Hugo, with interactive visualizers powered by TypeScript and WebGPU.

## Technology Stack

- **Static Site Generator**: Hugo
- **Styling**: Dart Sass (compiled to CSS via Hugo Pipes)
- **Scripting**: TypeScript
- **Linting & Formatting**: ESLint and Prettier
- **Documentation**: TypeDoc (for TypeScript components)
- **CI/CD**: GitHub Actions and GitHub Pages

## Directory Structure

- `content/`: Contains the Markdown files for the tutorial sections and lessons.
- `assets/ts/`: Contains the TypeScript source code for interactive WGSL visualizers and the site's frontend logic.
- `layouts/`: Contains Hugo HTML templates for rendering the site.
- `static/`: Contains public static assets that are served as-is (e.g., images, third-party libraries). Note: Do NOT manually modify the root `public/` directory, as it is auto-generated by Hugo during the build process.
- `.github/workflows/`: Contains the GitHub Actions used for continuous integration and deployment.

## Development Workflow

- **Install Dependencies**: Run `npm install` to install Node packages. Note: The `postinstall` script sets up a Git hook for PR checks.
- **Run Locally**: Use `npm run serve` (which runs `hugo serve`) to run the development server locally.
- **Linting**: Run `npm run lint` to lint the TypeScript files.
- **Formatting**: Run `npm run format` to run Prettier across the codebase.
- **Build**: Running `hugo --minify` will build the production-ready site to the `public/` directory.

## Publishing Process

The site uses automated GitHub Actions workflows for deployment:

1. **Production**: Merges or pushes to the `main` branch trigger the `deploy-main.yml` workflow, which builds the Hugo site and deploys it directly to the `gh-pages` branch for hosting on GitHub Pages.
2. **Pull Requests**: For PRs, the `build-pr.yml` and `deploy-pr.yml` workflows build a preview of the site and deploy it to a `pr-preview/pr-<NUMBER>` subfolder on the `gh-pages` branch, providing a preview link automatically on the PR.

## Agent Instructions

1. **TypeScript Edits**: Always run `npm run lint` and `npx tsc --noEmit` after modifying TypeScript code to verify correctness.
2. **Read-Only Output**: Do NOT manually modify the `public/` or `docs/` directories, as these are generated artifacts.
3. **Hugo Awareness**: If changing CSS or JS logic, keep in mind that Hugo pipes handles the asset bundling and transpilation from the `assets/` directory.
4. **Environment Constraints**: Do not put Git repositories (`.git/`) or sensitive configurations inside the `static/` directory as it is publicly served.
